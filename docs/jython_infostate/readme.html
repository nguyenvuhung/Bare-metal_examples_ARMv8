Jython State Display Example - ARM®DS-5™

This example illustrates the use of a Jython script in DS-5 Debugger to display the state (stack frame / execution context) of an application.
Note
For information on locating the example files and extracting them for use either on the command-line or in Eclipse, see the installation instructions.

Purpose and scope

This example illustrates the use of a Jython script in DS-5 Debugger to display the state (stack frame / execution context) of an application. It can be used with DS-5 Debugger connected to any ARM-based target, either bare-metal or Linux. It uses several DS-5 Debugger-specific Jython classes/services (such as getExecutionContext(), getProgramAddress(), getRegisterNames(), etc) to perform the operation. This example is intended to illustrate features of the DS-5 Debugger - it is not intended to be a tutorial on Jython.

Using the example

To use this example, first connect DS-5 Debugger to a target in the normal way, run some code, then stop the target at the point of interest. Then run the infostate.py Jython script in one of the following ways:
In the Command field of the Command view, enter the source command to run infostate.py, for example:
source jython_infostate/infostate.py
Alternatively, "drag and drop" the infostate.py file from the Project Explorer view into the Scripts view, then double-click on it to execute it. The Jython Script Parameters control in the Scripts view also allows you to specify parameters to be passed to a script.

It is also possible to trigger execution of a Jython script when a breakpoint is hit:
  1. Connect to a target, and set a breakpoint in the code in the normal way
  2. In the Breakpoints view, right-mouse-click on the breakpoint and select Properties
  3. In the On break, run script field, enter the name of the Jython (.py) script you wish to run. A Debugger (.ds) script could be specified here instead.
  4. Run the code, and when the breakpoint is hit, the specified script will execute.
